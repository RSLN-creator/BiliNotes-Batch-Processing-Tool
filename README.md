# BiliNote 批量收藏转写工具

批量调用 BiliNotes Docker 容器，对 B站收藏夹视频进行高通量笔记转写。支持 Bilishelf 浏览器插件导出的 JSON/CSV 格式。

## 1. 快速开始

1. 确保 **BiliNotes Docker 容器**已启动（默认地址 `http://localhost:3015`）
2. 双击 `启动.bat` 或命令行运行 `python app.py`
3. 浏览器打开 `http://localhost:18765`
4. 点击「导入收藏数据」选择从 Bilishelf 导出的 JSON 或 CSV 文件
5. 配置模型、转写引擎和笔记风格
6. 点击「处理全部待处理」批量处理，或勾选后「处理选中」

## 2. 前置要求

- **Python 3.8+**（安装时勾选"Add Python to PATH"）
- **BiliNotes Docker 容器** 已启动并运行在 `http://localhost:3015`
- **Bilishelf 浏览器插件** 导出的收藏数据（JSON 推荐，信息更全）

## 3. 界面布局

整个页面从上到下分为几个功能区块，可通过右下角悬浮目录按钮 `📑` 快速跳转。

### 3.1 模型与转写配置

- **供应商**：选择 LLM API 供应商（自动从 BiliNote 拉取）
- **LLM 模型**：选择具体推理模型，选择后自动检测是否支持多模态（截图/视频理解）
- **转写引擎**：ASR 语音识别引擎，支持 fast-whisper、mlx-whisper 等本地引擎及必剪、快手等在线引擎
- **转写模型**：whisper 模型尺寸（tiny/base/small/medium/large），仅本地引擎可选
- **音频质量**：fast（快速）/ medium（均衡）/ slow（高保真）

### 3.2 笔记风格与格式

**风格**：精简、详细、教程、学术、小红书、生活向、任务导向、商业风格、会议纪要

**格式选项**：
- 目录 — 生成结构化目录
- AI 总结 — 引言与多维度总结
- 截图 — 内嵌关键帧截图（需多模态模型）
- 跳转时间轴 — 绝对时间戳秒级跳转（不建议开启：影响md目录）

**保存内容**：Markdown 笔记、原文字幕(TXT)、完整 JSON、思维导图

**视频理解**：启用视觉帧大模型分析，可调节截图间隔（秒）和网格布局（行×列），需多模态模型支持

### 3.3 执行控制台

- **输出目录**：本地文件落盘路径，默认 `./output`
- **处理全部待处理**：提交所有未处理视频（自动跳过已处理）
- **处理选中**：仅处理勾选的视频（也自动跳过已处理）
- **紧急熔断**：停止批量处理（完成当前任务后结束）

**运行监控**：批量处理期间每 30 秒自动检测：

- **BiliNote 服务存活**：通过 `/api/sys_health` 检查后端进程是否正常（backend/ffmpeg/db/whisper_model），崩溃则终止任务
- **任务是否在运行**：通过 Docker 检查容器内任务状态文件是否存在，文件丢失则判定任务异常终止

### 3.4 视频列表

- **双视图切换**：表格视图（经典多维网格）↔ 卡片视图（视听卡片透视）
- **状态筛选**：全部 / 待处理 / 已处理 / 失败 / AI匹配 / 多P视频
- **文件夹筛选**：按收藏夹分组过滤
- **搜索**：支持 BV号、UP主名、UP主 UID、标题模糊搜索
- **分页**：每页 30 条，避免大量数据加载卡顿
- **已处理跳过**：批量处理时自动跳过已处理视频

每行视频包含：BV号、标题（可点击跳转B站）、UP主（可点击跳转B站空间）、处理状态、操作按钮。

### 3.5 处理进度

实时显示：总数、成功、失败、跳过、当前处理中，以及进度条和终端日志。

### 3.6 文件导出

一键按模态重组拷贝：汇聚全部 `.md` 笔记 / `.txt` 字幕 / `.json` 数据 / 思维导图到指定目录。

## 4. 卡片视图

卡片模式下每个视频展示为独立卡片，包含：

- **封面图**：B站视频封面（自动处理防盗链），无封面时隐藏
- **状态标签**：浮动在封面左上角（已处理=绿 / 失败=红 / 待处理=灰）
- **标题**：最多3行，超出省略
- **标签行**：UP主（可点击）→ BV号 → P标签 → AI字幕标记 → 文件夹（可点击筛选）
- **简介**：有简介时显示 `📝 简介` 蓝色标签，点击弹出气泡查看完整描述
- **操作**：打开笔记 / 文件夹 / 重处理 / 强制重转（AI字幕时）
- **多P视频**：显示 P 数选择下拉框

## 5. 文件夹分组筛选

- 自动从导入数据中提取所有收藏夹分组
- 下拉菜单位于视图切换按钮左侧
- 选择分组后视频列表仅显示该分组内容
- 卡片视图点击文件夹标签也可触发筛选
- 支持 Bilishelf 两种导出格式：`folderId` 字段和 `folderItems` 映射表

## 6. 多P视频支持

- 自动识别分P数 > 1 的视频，BV号旁显示橙色 `NP` 标签
- **🔍 检测多P**：通过 B站 API 查询真实分P数，自动刷新 pageCount
- **多P筛选芯片**：待处理 / 部分处理 / 全处理，一键过滤
- **P 数选择面板**：点击多P标签弹出勾选框，支持全选/全不选/选区
- **批量多P处理**：「处理全部待处理多P」自动处理所有未完成分P，**自动跳过已处理分P**
- 提交时 URL 自动追加 `?p=N`
- 笔记保存路径加 ` - PN` 后缀，不同分P独立目录互不覆盖
- **多P笔记打开**：已处理的多P视频点击"打开笔记"弹出 P 选择器
- **分P 字幕修复**：BiliNote 后端已修复分P 字幕获取（从 `pages` 数组取对应 cid）

## 7. 同步到 BiliNotes 前端

批量工具提交的任务存储在 BiliNote 后端，但不会自动显示在前端历史列表中。同步步骤：

1. 点击顶栏「🔄 同步」按钮，打开同步窗口
2. 系统自动生成注入脚本并按**每批 200 条**分批显示
3. 逐批点击「📋 复制」，粘贴到 BiliNotes 控制台 (F12→Console) 执行
4. 全部批次执行完毕后，手动刷新 BiliNotes 页面 (F5)

**同步逻辑**：
- 优先从 BiliNote API 获取笔记内容，API 缓存丢失时自动读取本地 `完整.json` 兜底
- 按 **video_url 去重**：同步前先清掉 IndexedDB 中同 URL 的旧记录，再插入新记录。强制删除重跑后不会残留旧笔记
- 每批独立注入脚本，防止大量数据粘贴卡死浏览器
- 仅同步状态为 SUCCESS 且有实际笔记内容的任务

## 8. 输出目录结构

```
output/
├── _已处理.json                    # 处理记录（checkpoint，key=bvid|pN复合键）
├── _已检测分P.json                 # 多P检测缓存（检测结果+已处理P号）
├── 0-竞赛！/
│   ├── BVxxxxx - 视频标题/
│   │   ├── 笔记.md
│   │   ├── 原文.txt
│   │   ├── 完整.json
│   │   └── 思维导图.svg            # 自动从markdown标题生成
│   └── BVxxxxx - 多P视频标题 - P2/
│       └── ...
├── 3-绘画教程/
│   └── ...
└── 5-AI转记/
    └── ...
```

**导出**：一键导出到目标目录，按视频分组合并。多P 视频归入同一文件夹，文件名加 `- P{n}` 区分。

## 9. 强制本地转写

勾选输出目录旁的「🎙️ 强制本地转写」后，提交任务时将跳过 B站 AI 字幕，直接下载音频并使用本地 Whisper 模型（fast-whisper）进行转写。转写结果更准确，但速度较慢且需要 GPU 支持。

- 启用后，checkpoint 记录会标记 `skip_subtitle: true`
- 视频列表自动显示 `🎙️ 本地` 标签（卡片和表格视图）
- 支持「🎙️ 本地转写」筛选芯片，一键过滤所有本地转写视频
- **注意**：需要 BiliNote 后端升级（`routers/note.py` + `services/note.py` 的 `skip_subtitle` 支持）

## 10. 强制删除缓存

每个已处理视频（包括失败和待处理）都有「🗑 强制删除」按钮。点击确认后将：

1. 删除 BiliNote 容器内的任务缓存文件（transcript/audio/result/status）
2. 删除本地 `output/` 目录下的对应笔记文件夹
3. 清除 checkpoint 记录和 `_已检测分P.json` 中的处理状态
4. 视频回到待处理状态，可重新提交

适用于：重新转写、更换模型、修复错误笔记等场景。

## 11. 思维导图

保存时自动从 Markdown 笔记的标题层级（`#` / `##` / `###`）生成水平树形 SVG 思维导图，保存在输出目录中。

## 13. 常见问题

### 13.1 封面图不显示？

B站 CDN 有防盗链机制（检查 HTTP Referer），页面已添加全局 `<meta name="referrer" content="no-referrer">` 及图片标签 `referrerpolicy="no-referrer"` 处理。如仍不显示请检查网络或B站 CDN 状态。

### 13.2 导入 JSON 后所有视频显示"未分类"？

部分 Bilishelf 导出格式（如全量导出）的视频不带 `folderId` 字段，而是通过 `folderItems` 映射表关联。工具已兼容两种格式，如仍异常请检查 JSON 文件结构。

### 13.3 🔄 重新处理 和 🗑 强制删除 有什么区别？

| | 🔄 重新处理 | 🗑 强制删除 |
|---|---|---|
| 清 checkpoint | ✅ | ✅ |
| 删 BiliNote 容器缓存 | ❌ | ✅ |
| 删本地 output 文件 | ❌ | ✅ |
| 清多P 处理记录 | ❌ | ✅ |
| 重跑后笔记内容 | **和之前一样** | **可换模型/本地转写** |

**🔄**：仅清除处理记录，让视频回到待处理状态。每次提交都生成新 task_id，旧缓存不会被复用，但 B站 字幕内容没变 → 笔记和之前完全一样。适合「网络波动失败，重试一下」。

**🗑**：彻底清空所有缓存和本地文件。配合「🎙️ 强制本地转写」可以从 AI 字幕切换到本地 Whisper 转写，得到不同的笔记。适合「换模型重跑」「修复错乱文件」。

### 13.4 多P视频怎么处理？

使用「🔍 检测多P」查询真实分P数 → 点击多P标签勾选要处理的分P → 「处理全部待处理多P」自动跳过已完成分P。每个分P独立保存，目录名加 ` - PN` 后缀区分。

### 13.5 强制本地转写和 B站 AI 字幕有什么区别？

B站 AI 字幕：速度快，直接复用 B站已有的 AI 生成字幕，不消耗 GPU。
本地 Whisper：下载音频后用本地模型转写，更准确，但需要 GPU 且速度较慢。勾选「🎙️ 强制本地转写」启用。

### 13.6 端口 8765 无法启动？

Windows 可能将 8765 划入动态端口池。工具默认端口已改为 **18765**。如仍需自定义：`set GUI_PORT=自定义端口 && python app.py`

### 10.5 换电脑或重装后如何恢复？

保留 `output/` 目录（含 `_已处理.json`），在新环境重新导入相同收藏文件即可恢复所有处理状态。封面、UP主、分P 等元数据会自动从导入文件中回填。

### 10.6 同步到 BiliNotes 后历史列表没变化？

执行注入脚本后需手动刷新 BiliNotes 页面 (F5)。同步去重逻辑确保已有任务不会丢失。

### 10.7 多模态模型检测不准确？

工具通过模型名称关键词匹配判断是否支持视觉（vision/vl/4o/claude/gemini/qwen 等）。如果误判，可手动在 BiliNote 中查看模型能力或提交 issue。
