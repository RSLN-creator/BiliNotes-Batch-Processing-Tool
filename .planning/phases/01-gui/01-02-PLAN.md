  <action>
    在 `app.py` 中添加以下内容：

    1. **`POST /api/parse-file` 端点**：
       - 接收 JSON body: `{ content: string, filename: string, fileType: "json"|"csv" }`
       - 根据 `fileType` 调用对应的解析函数
       - 成功返回 `{ videos: [...], total: number }`
       - 失败（解析异常）返回 400 + `{ error: "错误描述" }`

    2. **`parse_bilishelf_json(content: str) -> list[dict]`**：
       - 使用 `json.loads` 解析
       - 遍历 `data["folders"]`，每个 folder 的 `title` 和 `videos[]`
       - 每个 video 提取 `bvid`, `title`, `url`, `folder`（收藏夹名）
       - 返回 `[{bvid, title, url, folder}, ...]`

    3. **`parse_bilishelf_csv(content: str) -> list[dict]`**：
       - 使用 `csv.DictReader` 和 `io.StringIO` 解析
       - 定义 `COLUMN_MAP` 支持别名映射（Bilishelf CSV 列名不确定，Pitfall 1）：
         - `bvid`: `['bvid', 'bvid', 'BV号', 'BV', 'avid', 'aid']`
         - `title`: `['title', '标题', '视频标题', 'name']`
         - `folder`: `['folder', 'folders', '收藏夹', 'folder_title', 'folder_name']`
         - `url`: `['url', '链接', 'video_url', 'page_url']`
       - 对每行遍历 `COLUMN_MAP` 找到第一个匹配的列名；缺少的字段设为空字符串 `''`
       - 返回 `[{bvid, title, folder, url}, ...]`

    4. **注意事项**：
       - 函数定义放在文件末尾（`parse_file()` 路由之后）
       - 参考 RESEARCH.md 第 298-362 行的具体实现结构
       - 不添加内容大小限制（Phase 3 处理 EDGE 边界）
       - 不要修改 `app.py` 中已有的 "/" 路由和启动代码
  </action>

    在 `static/js/app.js` 中创建完整的前端交互逻辑（D-03 锁定：事件驱动 + DOM 操作模式，原生 JS 无框架）：

    1. **状态对象** `state`：
       - `videos: []` — 当前加载的视频列表
       - `selectedFile: null` — 当前选择的文件

    2. **DOM 引用**：使用 `const $ = (id) => document.getElementById(id)` 快捷函数引用：
       - `fileInput = $('file-input')`, `selectFileBtn = $('select-file-btn')`, `fileName = $('file-name')`
       - `providerSelect = $('provider-select')`, `modelSelect = $('model-select')`
       - `videoList = $('video-list')`, `videoCount = $('video-count')`
       - `banner = $('banner')`

    3. **事件绑定**（在 `DOMContentLoaded` 中）：
       - `selectFileBtn.addEventListener('click', () => fileInput.click())` — 按钮触发隐藏 input
       - `fileInput.addEventListener('change', onFileSelected)` — 文件选择后触发

    4. **`async function onFileSelected(event)`**：
       - 获取 `event.target.files[0]`，存入 `state.selectedFile`
       - 显示文件名到 `fileName.textContent`
       - 调用 `await loadFile(file)`

    5. **`async function loadFile(file)`**：
       - `const text = await file.text()` — FileReader 替代方案（Blob.text()）
       - 根据 `file.name` 后缀判断 `fileType: 'csv' 或 'json'`
       - `fetch('/api/parse-file', { method: 'POST', headers: {'Content-Type': 'application/json'}, body: JSON.stringify({content: text, filename: file.name, fileType}) })`
       - 响应非 200：`showBanner('文件解析失败：' + err, 'error')`；return
       - 成功：`state.videos = data.videos; renderVideoList(data.videos); hideBanner()`
       - 异常 catch：`showBanner('文件读取失败：' + err.message, 'error')`

    6. **`function renderVideoList(videos)`**（D-06：错误用横幅，不弹窗）：
       - 空列表：显示 `<div class="empty-state">文件中未找到视频数据</div>`
       - 非空：按 `v.folder` 分组（`|| '未分类'`）
       - 每个收藏夹渲染一个 `<div class="folder-group">`：
         - `<h3 class="folder-title">收藏夹名 (N)</h3>`
         - `<table class="video-table">` 含表头 `BV号 | 标题 | 状态`
         - 每行：`<tr>` → `<td class="bvid">bvid</td>` + `<td>title</td>` + `<td><span class="status-badge status-pending">待处理</span></td>`
       - 更新 `videoCount.textContent = '共 N 个视频'`
       - 所有用户输入使用 `escapeHtml()` 处理（防止 XSS）

    7. **辅助函数**：
       - `showBanner(message, type = 'warning')` — 设置 `banner.textContent` 和 `banner.className`
       - `hideBanner()` — `banner.className = 'banner hidden'`
       - `escapeHtml(str)` — 用 `textContent` 设置后返回 `innerHTML`（防止 XSS）

    8. **注意事项**：
       - 使用 `file.text()` Promise API（所有现代浏览器支持），不手动创建 FileReader
       - 所有 `fetch` 必须包裹 try/catch
       - 不在此阶段实现供应商/模型加载（Plan 03 添加）
       - 遵循 D-04：只在用户选择文件后检测连接（loadFile 成功后调用 API），不阻塞启动
       - 当前 `loadFile` 末尾不调用 `loadProviders()`（Plan 03 添加该逻辑）
       - 遵循 D-03：全部使用原生 DOM API，无框架、无第三方 JS 库
